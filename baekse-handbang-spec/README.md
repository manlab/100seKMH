# 백세한방병원 홈페이지 — 빌드 명세서

> 클라이언트: 백세한방병원 (개원 예정)
> 빌드 대상: 반응형 공식 홈페이지
> 기술 스택: **Next.js 14 (App Router) + TypeScript + Tailwind CSS**
> 작성일: 2026-05-01

---

## 1. 이 폴더의 목적

이 폴더는 **AI 코딩 도구(Claude Code / Cursor / Windsurf 등)에 그대로 던져 넣어 빌드를 시작하기 위한 사양서 패키지**입니다. 디자이너의 시안을 픽셀 단위로 옮기는 것이 아니라, 토큰 → 컴포넌트 → 페이지 → 콘텐츠 순으로 일관된 룩앤필을 자동 생성하도록 설계되어 있습니다.

홈페이지 본문은 아직 개원 전이라는 점을 고려해 **틀(레이아웃·디자인 시스템·라우트·섹션 구성)을 완성도 있게 잡고, 카피·이미지·실제 의료진 정보는 마커로 비워두는 전략**입니다. 개원 시점에 원장님 사진·인사말·연락처만 채우면 그대로 라이브가 가능합니다.

## 2. 파일 구조

```
baekse-handbang-spec/
├── README.md              ← (지금 보고 있는 파일) 인덱스 / 사용 가이드
├── 01-design-system.md    ← 컬러·타이포·스페이싱·컴포넌트 토큰
├── 02-sitemap-ia.md       ← 사이트맵, 라우트, 네비게이션, 페이지 목록
├── 03-content-spec.md     ← 페이지별 섹션 구성과 카피 가이드
├── 04-build-prompt.md     ← AI 코딩 도구용 마스터 빌드 프롬프트
├── tailwind.config.ts     ← 실제 코드에 복붙 가능한 Tailwind 토큰
└── globals.css            ← CSS 변수, 폰트, 베이스 스타일
```

각 문서는 독립적으로 읽을 수 있지만, 새로 빌드를 시작할 때는 다음 순서로 사용하세요.

1. `04-build-prompt.md` 전체를 코딩 에이전트에게 시스템 프롬프트로 전달
2. `01-design-system.md` + `tailwind.config.ts` + `globals.css` 를 같이 첨부
3. `02-sitemap-ia.md` 로 라우트 골격 생성 요청
4. `03-content-spec.md` 로 페이지별 섹션을 채워 넣기

## 3. 디자인 컨셉 한 줄 요약

> **로고의 딥네이비(#143A6B)를 메인으로, 청록 삼각형(#4A8E9C)을 액센트로. "100"이라는 숫자가 가진 묵직한 신뢰감과 한방의 따뜻한 정밀함을 같이 담는다.**

- **신뢰**: 의료 사이트의 기본기 — 충분한 여백, 또렷한 위계, 무리 없는 색 대비.
- **전통**: 한방 특유의 절제된 동양적 감성을 라인·그림자·아이콘에 반영.
- **현대**: 평일·토요일 진료, 모바일 우선, 빠른 진료 예약/상담 진입점.

## 4. 진료 카테고리 (백세한방병원 버전)

미래한방병원(참고 사이트)의 6개 진료 카테고리 골격은 유지하되, **한방 중심**으로 명칭과 콘텐츠를 재구성했습니다. 양방 협진 관련 표현은 축소하거나 "필요시 협진" 정도로 톤다운.

| 카테고리 | 클리닉 | 비고 |
|---|---|---|
| 1. 백세한방병원 | 인사말, 의료진, 시설, 한약 안내, 진료시간, 오시는길 | 병원소개 메뉴 |
| 2. 척추·관절 | 디스크, 협착증, 오십견, 무릎관절통, 스포츠상해 | 통증재활 |
| 3. 추나·체형교정 | 척추측만증, 일자목/거북목, 골반/휜다리, 산후교정 | 자세교정 |
| 4. 교통사고 한방치료 | 후유증, 자보 절차, 입원 시스템 | 자동차보험 |
| 5. 면역·만성 | 비염/천식, 만성피로, 수면/두통, 수족냉증 | 보약·보양 |
| 6. 여성·소아 | 산후조리, 생리통, 갱년기, 성장/총명탕 | 여성·아이 |
| 7. 커뮤니티 | 공지, 온라인상담, FAQ, 비급여, 서류발급 | CS |

(상세 라우트는 `02-sitemap-ia.md` 참고)

## 5. 빠른 인수인계용 체크리스트

개발자가 합류했을 때 가장 먼저 확인해야 할 항목:

- 로고 SVG 원본 받기 (아직 PNG만 있음 → SVG 변환 필요)
- 의료진 인물 사진 (정면 무배경, 1200x1500 이상 권장)
- 시설 사진 18~22장 (가로 4:3 또는 3:2)
- 병원 정확한 주소·전화번호·사업자등록번호
- 카카오맵 또는 네이버지도 임베드용 좌표
- 도메인 (예: baeksehospital.kr) 및 호스팅
- 진료시간 확정 (평일/토요일/일요일 휴진/공휴일 별도 공지/점심시간)
- 비급여 항목 표 (보건복지부 고시 양식)
- 개인정보처리방침·이용약관·환자권리장전 (병원장 명의 검토)
- 네이버 블로그·인스타·유튜브 등 외부 채널 URL

위 항목이 비어있어도 사이트는 빌드되도록 `// TODO(client-asset)` 마커로 처리해 두었습니다.
